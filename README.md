# Алгоритмы и структуры данных. Set 3. Задача A3b. Экспериментальные данные
Данный репозиторий содержит результаты экспериментального исследования
производительности двух алгоритмов сортировки целочисленных массивов:

- **Quick Sort** — классическая быстрая сортировка с разбиением массива  
  (опорный элемент выбирается случайным образом)
- **Intro Sort** — гибридный алгоритм (Quick Sort + Heap Sort + Insertion Sort),  
  автоматически переключающийся:
  - на **Insertion Sort** при размере подмассива < 16
  - на **Heap Sort** при глубокой рекурсии (*глубина > 2 × log₂(N)*)

---

## Типы исследуемых входных данных

Для анализа эффективности алгоритмов использовались три вида входных массивов:

| Тип массива | Описание |
|------------|-----------|
| **Random** | Случайный массив |
| **Inverted** | Полностью отсортирован в обратном порядке |
| **Almost Sorted** | Почти отсортирован, содержит небольшие отклонения |

Эксперименты проводились для массивов размера **от 500 до 100000**  
с шагом **1000 элементов**.

Каждый замер выполнялся **7 раз**, после чего вычислялось среднее значение.

## Формат данных в CSV

Каждая строка описывает среднее время выполнения алгоритмов для конкретного размера массива `N`.

| Поле | Тип данных | Описание |
|------|------------|----------|
| **N** | int | Размер массива |
| **Quick_Random** | int | Время Quick Sort на случайных данных |
| **Intro_Random** | int | Время Intro Sort на случайных данных |
| **Quick_Inverted** | int | Время Quick Sort на инвертированных данных |
| **Intro_Inverted** | int | Время Intro Sort на инвертированных данных |
| **Quick_Almost** | int | Время Quick Sort на почти отсортированных данных |
| **Intro_Almost** | int | Время Intro Sort на почти отсортированных данных |
